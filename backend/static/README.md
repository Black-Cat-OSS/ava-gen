# Директория статического контента

Эта директория содержит статические файлы, которые раздаются через NestJS приложение из корня `/`.

## Назначение

Директория `static/` предназначена для размещения собранного React приложения (или любого другого frontend приложения).

## Размещение React приложения

### Шаг 1: Сборка React приложения

```bash
cd frontend
npm run build
```

### Шаг 2: Копирование файлов

Скопируйте содержимое `frontend/dist/` (или `frontend/build/`) в эту директорию:

```bash
# Из корня проекта
cp -r frontend/dist/* backend/static/
```

### Шаг 3: Сборка backend

```bash
cd backend
npm run build
```

## Доступ к файлам

Все файлы из этой директории доступны из корня:

- `http://localhost:3000/` - главная страница (index.html)
- `http://localhost:3000/assets/...` - ресурсы приложения
- Любые другие файлы из этой директории

**Важно:** Маршруты `/api/*` и `/swagger/*` зарезервированы для API и документации.

## Структура после размещения React приложения

```
static/
├── index.html           # Главная страница React приложения
├── assets/              # Ресурсы (JS, CSS, изображения)
│   ├── index-*.js
│   ├── index-*.css
│   └── ...
└── README.md            # Этот файл
```

## Примечания

- Файлы автоматически копируются в `dist/static/` при сборке backend
- В режиме разработки изменения отслеживаются автоматически
- React Router будет работать корректно
- Подробная документация: `backend/docs/STATIC_CONTENT.md`
